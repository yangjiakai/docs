next-forge 是一个 monorepo（单体仓库），这意味着它在一个代码仓库中包含多个软件包。这是现代 Web 应用程序的常见模式，因为它允许你在应用程序的不同部分之间共享代码，并统一管理它们。

这个 monorepo 由 [Turborepo](https://turbo.build/repo) 管理，这是一个用于管理单体仓库的工具。它提供了一种简单的方式来管理单个代码仓库中的多个软件包，并专门为现代 Web 应用程序设计。

该 monorepo 包含以下应用程序：

| 应用程序    | 描述                                                                          | 端口 | 推荐 URL        |
| ----------- | ----------------------------------------------------------------------------- | ---- | --------------- |
| `api`       | 包含设计用于独立于主应用程序运行的无服务器函数，例如 webhooks 和定时任务。    | 3002 | `api.acme.com`  |
| `app`       | 主应用程序，采用 [shadcn/ui](https://ui.shadcn.com/) 模板。                   | 3000 | `app.acme.com`  |
| `docs`      | 文档站点，包含应用程序的文档，例如指南和教程。                                | 3004 | `docs.acme.com` |
| `email`     | 来自 [react.email](https://react.email/) 的邮件预览服务器。                   | 3003 |                 |
| `storybook` | Storybook，包含应用程序的组件展示和文档。                                     | 6006 |                 |
| `studio`    | [Prisma Studio](https://www.prisma.io/studio)，这是一个数据库的图形化编辑器。 | 3005 |                 |
| `web`       | 网站，采用 [twblocks](https://www.twblocks.com/) 模板。                       | 3001 | `acme.com`      |

虽然你可以选择在任何子域名上运行这些应用程序，但上面列出了推荐的子域名。记得在[环境变量](/env)中添加它们，包括 `NEXT_PUBLIC_APP_URL`、`NEXT_PUBLIC_WEB_URL` 和 `NEXT_PUBLIC_DOCS_URL`。
