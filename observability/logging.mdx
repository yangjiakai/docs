日志功能通过一个简单的包装器抽象出来，提供了一个一致的日志接口，适用于所有环境。

## 使用方法

要使用此日志设置，只需导入并使用 `log` 对象即可。它与 `console` 对象共享相同的接口，因此可以在代码库中将 `console` 替换为 `log`。

```tsx page.tsx
import { log } from '@repo/observability/log';

log.info('这是一条信息日志');
```

## 日志级别

日志级别按照以下顺序从低到高排列：

1. debug
2. info
3. warn
4. error

你可以通过设置 `LOG_LEVEL` 环境变量来控制日志级别。

## 日志记录原理

在开发环境中，日志输出到控制台以便于调试。在生产环境中，日志发送到 BetterStack Logs 进行搜索、过滤和分析。
